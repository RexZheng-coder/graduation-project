# Artifacts

This directory stores generated outputs created during model training, serialization, debugging, and experiment tracking.

It is the part of the repository that answers, "What did the project produce while it was being built?"

## Contents

- `model-exports/` contains saved model files and related metadata such as hash listings.
- `training-checkpoints/` contains checkpoint files produced during training runs.
- `tracking/` contains MLflow and Lightning tracking outputs collected from experiments.

## How To Read This Directory

- Use `model-exports/` when you want concrete examples of serialized model outputs.
- Use `training-checkpoints/` when you want to inspect intermediate training states.
- Use `tracking/` when you want experiment metadata, logged runs, and training summaries.

## Why It Is Separate

These files are useful, but they are not the main codebase. Pulling them into a dedicated directory makes it much easier to separate:

- implementation code,
- reproducibility materials,
- and generated byproducts of experimentation.
