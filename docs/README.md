# Docs

This directory stores the reference material that explains the project from the thesis and documentation side.

It exists to separate explanatory material from code. The goal is to keep the engineering prototype in `Re2po/` focused on implementation while keeping paper-facing material and supporting references easy to find.

## Contents

- `paper/` contains the exported paper copy associated with the Re2po project.

## How To Use This Directory

- Read the paper first if you want the academic framing, evaluation language, and system motivation.
- Read `Re2po/README.md` first if you want the implementation-oriented view.
- Use both together when turning this work into portfolio material, because the paper gives the research story and `Re2po/` gives the engineering story.
