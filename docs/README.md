# Docs

This directory stores the reference material that explains the project from the planning, thesis, and documentation side.

It exists to separate explanatory material from code. The goal is to keep the engineering prototype in `Re2po/` focused on implementation while keeping paper-facing material and supporting references easy to find.

## Contents

- `paper/` contains the planning paper and the Re2po paper associated with this project.

## How To Use This Directory

- Read the planning paper first if you want the broader motivation and the earliest stage of the project.
- Read the Re2po paper next if you want the follow-on system formulation that came after the planning stage.
- Read `Re2po/README.md` after the two papers if you want the implementation-oriented view in chronological order.
- Use the papers together when turning this work into portfolio material, because they give the research story and `Re2po/` gives the engineering story.
